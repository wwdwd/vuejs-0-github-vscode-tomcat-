1.个人版本控制
环境：vscode + github 
  ①使用github在文件系统中建立一个工作空间
  ②使用vscode打开①中的工作空间，使用vscode中的版本控制工具即可
  扩展：vscode的版本工具中，发布功能 将新工程发布到github，发布之后方可推送。

  如果利用gitbash 先建立工作控件，可以直接使用 推送功能。拉取和同步 都将

  如果先在本地构建工程，再上传GITHUB
①我们在github上创建一个Repository，复制地址备用
②git bash进入新项目所在文件根目录
	git remote add origin https://github.com/xuanhun/vscode.git
	git pull origin master
③此时config文件中已经出现 链接地址
④vscode 发布项目，之后可以提交

3.GIT对于分支的操作，因为VSCODE 对于GIT的集成 只进行到对代码层次的提交，同步，没有涉及到 分支层次的合并和删除但依然可以新建和切换分支，因此需要使用GIT BASH 进行分支层次的操作。
GIT 分支： 查看分支 git branch -a
将test2.x分支合并到xm3.0分支的话，进入xm3.0分支运行git?merge test2.x命令
